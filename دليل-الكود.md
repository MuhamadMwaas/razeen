## 1. نظرة عامة والفكرة الأساسية

الموقع مبني بثلاث تقنيات فقط، **بدون أي إطار عمل** (No Bootstrap / No Tailwind /
No React):

| التقنية | الدور | الملف |
|---|---|---|
| **HTML5** | هيكل الصفحة (العظام) | `index.html` |
| **CSS3** | الشكل والتنسيق (الجلد والألوان) | `css/style.css` |
| **JavaScript** | الحركة والبيانات (العقل) | `js/main.js` |

### الفكرة الذهبية: «الموقع مدفوع بالبيانات» (Data-Driven)

بدل أن نكتب كل بطاقة منتج أو كل رابط في القائمة يدويًّا داخل HTML (تكرار ممل
وصعب التعديل)، اتّبعنا قاعدة:

> **البيانات تُكتب مرّة واحدة في مكان واحد** (كائن اسمه `DATA` داخل `main.js`)،
> **والـ JavaScript يأخذها ويبني الـ HTML تلقائيًّا.**

مثال للتوضيح: عندنا 4 بطاقات في قسم «المجموعات». لم نكتب 4 بطاقات في HTML، بل
كتبنا الـ HTML الفارغ مرّة واحدة (`<div id="collectionGrid"></div>`)، ووضعنا
بيانات البطاقات الأربعة في `DATA.collections`، ثم الـ JS يكرّر القالب 4 مرّات.

**النتيجة:** لو أردت تغيير اسم منتج أو إضافة رابط، تعدّل **سطر بيانات واحد** فقط،
ولا تلمس بنية HTML إطلاقًا.

## 3. كيف تُفتح الصفحة؟ (رحلة التحميل خطوة بخطوة)

عندما يفتح المستخدم الموقع، يحدث التالي بالترتيب:

1. **المتصفح يقرأ `index.html`** من أعلى لأسفل.
2. يصل إلى `<head>` فيُحمّل:
   - الخطوط من Google Fonts (Amiri, Tajawal, Playfair, Manrope).
   - ملف التنسيق `css/style.css`.
3. يبني **شجرة العناصر** (DOM) من وسوم `<body>`. في هذه المرحلة تكون بعض
   الحاويات **فارغة** عمدًا، مثل:
   ```html
   <ul class="nav-list" id="navList"><!-- JS --></ul>
   ```
   التعليق `<!-- JS -->` معناه: «هذا المكان سيملؤه الجافاسكربت لاحقًا».
4. يصل المتصفح إلى آخر سطر قبل `</body>`:
   ```html
   <script src="js/main.js"></script>
   ```
   فيُحمّل ويُشغّل ملف `main.js`.
5. **`main.js` يعمل الآن**: يقرأ كائن `DATA`، ويولّد HTML، ويحقنه داخل الحاويات
   الفارغة، ثم يفعّل التفاعلات (القائمة، الأزرار، تأثير الظهور...).

> **لماذا وُضع `<script>` في الأسفل وليس في `<head>`؟**
> حتى يكون الـ HTML قد بُني بالكامل قبل أن يعمل الجافاسكربت. لو شغّلناه في الأعلى،
> لن يجد العناصر التي يريد ملأها (لأنها لم تُرسم بعد) فيقع خطأ.

## 4. ملف `index.html` — الهيكل العظمي

### 4.1 وسم الصفحة واتجاه اللغة

```html
<html lang="ar" dir="rtl">
```

- `lang="ar"` ← لغة الصفحة عربية.
- `dir="rtl"` ← الاتجاه **من اليمين لليسار** (Right To Left). هذه أهم خاصّية في
  موقع عربي؛ تجعل النصوص والعناصر تبدأ من اليمين تلقائيًّا.

### 4.2 قسم `<head>`

```html
<meta charset="UTF-8" />                 ← يدعم الحروف العربية
<meta name="viewport" ... />             ← يجعل الموقع متجاوبًا مع الجوال
<title>رزين | أزياء محتشمة فاخرة</title>  ← عنوان التبويب
<meta name="description" ... />          ← وصف للموقع (مهم لجوجل/SEO)
<link href="...fonts..." />              ← تحميل الخطوط
<link rel="stylesheet" href="css/style.css" /> ← ربط ملف التنسيق
```

### 4.3 المفهوم الأهم: «الحاويات الفارغة»

في كل قسم تقريبًا سترى حاوية فيها `id` وتعليق `<!-- JS -->`. هذه هي «العلب
الفارغة» التي سيملؤها الجافاسكربت:

| الحاوية في HTML | يملؤها JS بـ |
|---|---|
| `<ul id="navList">` | روابط القائمة العلوية |
| `<div id="heroStats">` | أرقام الإحصائيات (4.9، +50...) |
| `<div id="heroTrust">` | شريط الثقة (ضمان الجودة، شحن سريع...) |
| `<div id="collectionGrid">` | بطاقات المجموعات الأربعة |
| `<div id="whyGrid">` | بطاقات «لماذا رزين» |
| `<div id="worldsGrid">` | بطاقات «عوالم رزين» (الأقواس) |
| `<div id="productRail">` | بطاقات المنتجات |
| `<div id="filterTypes">` / `id="filterColors"` | خيارات الفلترة |
| `<div id="specialSteps">` | خطوات «الطلب الخاص» |
| `<div id="footerGrid">` | أعمدة الفوتر |

> **القاعدة:** `id` فريد لكل عنصر في الصفحة، ويُستخدم كـ «عنوان» يلتقطه الجافاسكربت
> عبر `document.getElementById("navList")`.

### 4.4 الأجزاء الثابتة مقابل الأجزاء المُولّدة

ليس كل شيء يولّده JS. بعض النصوص مكتوبة مباشرة في HTML لأنها لا تتكرّر، مثل عنوان
الهيرو:

```html
<h1 class="hero-title">الأناقة</h1>
<p class="hero-lead">الوقار... جمالٌ لا يحتاج إلى لفت الأنظار</p>
```

**القاعدة العملية:**
- نصّ يظهر **مرّة واحدة** وثابت ← يُكتب مباشرة في `index.html`.
- عناصر **متكرّرة** (قوائم، بطاقات) ← تُكتب بياناتها في `DATA` ويولّدها `main.js`.

### 4.5 خاصّية `data-icon` (الأيقونات)

سترى أزرارًا مثل:

```html
<button class="icon-btn" aria-label="حقيبة التسوق" data-icon="bag"></button>
```

الزر فارغ، لكن فيه `data-icon="bag"`. الجافاسكربت يبحث عن كل عنصر فيه `data-icon`
ويضع داخله أيقونة SVG المناسبة (هنا أيقونة الحقيبة). شرحها التفصيلي في القسم 6.

### 4.6 الـ SVG الخاص بأقواس «عوالم رزين»

في نهاية قسم العوالم يوجد:

```html
<svg width="0" height="0" ...>
  <defs>
    <clipPath id="worldArch" clipPathUnits="objectBoundingBox">
      <path d="M0,1 L0,0.28 Q0.02,0.10 0.5,0 Q0.98,0.10 1,0.28 L1,1 Z" />
    </clipPath>
  </defs>
</svg>
```

هذا ليس صورة تظهر، بل **«قالب قَصّ»** (clip-path) على شكل قوس مدبّب. الـ CSS
يستخدمه (`clip-path: url(#worldArch)`) ليقصّ صور البطاقات على شكل القوس
الإسلامي. حجمه `0×0` لأنه تعريف فقط وليس عنصرًا مرئيًّا.

## 5. ملف `main.js` — العقل المدبّر

سنفكّك الملف إلى أجزاء.

### 5.1 الغلاف الخارجي (IIFE) و `"use strict"`

```js
(function () {
  "use strict";
  // ... كل الكود هنا ...
})();
```

- هذا الشكل `(function(){ ... })();` يُسمّى **IIFE** (دالة تُنفَّذ فور تعريفها).
- **فائدته:** كل المتغيّرات التي نعرّفها بالداخل (مثل `DATA`, `icon`...) تبقى
  **محبوسة** داخل هذه الدالة ولا «تتسرّب» إلى باقي الصفحة، فلا تتعارض مع أي كود
  آخر. يشبه وضع كل أدواتك في صندوق مغلق.
- `"use strict"` ← يفعّل «الوضع الصارم» الذي يجعل الجافاسكربت ينبّهك على الأخطاء
  بدل تجاهلها بصمت.

### 5.2 مكتبة الأيقونات `I` ودالة `icon()`

```js
const I = {
  bag: '<svg ...>...</svg>',
  user: '<svg ...>...</svg>',
  menu: '<svg ...>...</svg>',
  // ... عشرات الأيقونات
};
const icon = (n) => I[n] || "";
```

- `I` كائن (object) كل مفتاح فيه اسم أيقونة، وقيمته كود SVG كنصّ.
- `icon("bag")` ترجع كود أيقونة الحقيبة. لو طلبت اسمًا غير موجود، ترجع نصًّا
  فارغًا `""` (بفضل `|| ""`) بدل أن تتعطّل.
- **لماذا نخزّن SVG كنصّ؟** حتى نستطيع حقنه بسهولة داخل قوالب HTML لاحقًا.

> الأيقونات كلها تستخدم `stroke="currentColor"`، أي أنها **تأخذ لون النصّ المحيط
> بها تلقائيًّا** من الـ CSS. فلو غيّرت لون العنصر في CSS، تتغيّر الأيقونة معه.

### 5.3 كائن البيانات `DATA` — ⭐ أهمّ جزء للتعديل

هذا هو «لوحة التحكّم» في الموقع. كل المعلومات القابلة للتغيير موجودة هنا.

```js
const DATA = {
  nav: [ ... ],            // روابط القائمة العلوية
  stats: [ ... ],          // أرقام الإحصائيات في الهيرو
  trust: [ ... ],          // شريط الثقة (ضمان، شحن...)
  collections: [ ... ],    // بطاقات قسم المجموعات
  why: [ ... ],            // بطاقات «لماذا رزين»
  worlds: [ ... ],         // بطاقات «عوالم رزين»
  products: [ ... ],       // المنتجات
  filterTypes: [ ... ],    // خيارات فلتر النوع
  filterColors: [ ... ],   // خيارات فلتر اللون
  specialSteps: [ ... ],   // خطوات «الطلب الخاص»
  footer: [ ... ],         // أعمدة الفوتر
};
```

**كيف تقرأ هذا التركيب؟**

كل مفتاح قيمته **مصفوفة `[]`** (قائمة)، وكل عنصر في القائمة **كائن `{}`** (بطاقة
بيانات). مثال:

```js
collections: [
  { img: "ph-abaya", title: "عبايات", l1: "تصاميم أنيقة", l2: "لمختلف المناسبات" },
  { img: "ph-khimar", title: "خُمُر",   l1: "راحة وستر",   l2: "بخامات فاخرة" },
  // ...
],
```

- المصفوفة `[ ... ]` = «قائمة البطاقات».
- كل `{ ... }` = «بطاقة واحدة» لها خصائص:
  - `img` ← اسم الصورة.
  - `title` ← العنوان.
  - `l1` و `l2` ← سطرا الوصف.

لتغيير عنوان أول بطاقة من «عبايات» إلى «عبايات سهرة»، تعدّل `title` في أول كائن.
بهذه البساطة.

#### معاني الاختصارات في البيانات

| المفتاح | معناه | يظهر في |
|---|---|---|
| `label` | نصّ الرابط | القائمة، الفلاتر |
| `href` | وجهة الرابط (`#home`...) | القائمة |
| `active` | هل هو الرابط النشط؟ (`true`) | القائمة |
| `num` / `label` | الرقم ووصفه | الإحصائيات |
| `ic` | اسم الأيقونة (من مكتبة `I`) | الثقة، لماذا، الخطوات |
| `t` / `s` | عنوان / وصف فرعي | الثقة |
| `t` / `d` | عنوان / وصف | لماذا رزين |
| `n` | الرقم (01، 02...) | لماذا رزين |
| `title` / `sub` | عنوان / وصف | العوالم |
| `price` | السعر | المنتجات |
| `dots` | ألوان المنتج (قائمة أكواد لون) | المنتجات |
| `isNew` | شارة «جديد» (`true`) | المنتجات |

### 5.4 الدوال المساعدة (Helpers)

```js
const img = (name) => `assets/images/${name}.jpg`;
const COVER = "assets/cover.png";
const el = (id) => document.getElementById(id);
const setHTML = (id, html) => { const n = el(id); if (n) n.innerHTML = html; };
```

| الأداة | تفعل | مثال |
|---|---|---|
| `img("ph-abaya")` | تبني مسار صورة كامل | يرجع `"assets/images/ph-abaya.jpg"` |
| `COVER` | مسار صورة الغلاف المشتركة | `"assets/cover.png"` |
| `el("navList")` | تجلب عنصرًا بالـ id | يرجع عنصر `<ul>` |
| `setHTML("navList", "...")` | تضع HTML داخل عنصر | تملأ القائمة |

`setHTML` هي **بطل الحقن**: تأخذ `id` ونصّ HTML، تجد العنصر، وتضع النصّ بداخله
عبر `innerHTML`. وفيها حماية `if (n)`: لو لم يوجد العنصر، لا تفعل شيئًا (بدل أن
تتعطّل).

## 6. آلية الحقن: كيف تتحوّل البيانات إلى HTML؟

هذا قلب الموضوع. سنفهمه عبر ثلاثة مفاهيم، ثم مثال كامل.

### 6.1 المفهوم الأول: «القالب النصّي» (Template Literal)

النصّ المحاط بعلامة **`` ` ``** (backtick، وليس علامة اقتباس عادية) يُسمّى
template literal، وميزته أنه يسمح بإدخال متغيّرات بداخله عبر `${...}`:

```js
const name = "رزين";
const html = `<h1>مرحبًا ${name}</h1>`;
// النتيجة: "<h1>مرحبًا رزين</h1>"
```

- النصّ بين `` ` `` يمكن أن يمتدّ على عدّة أسطر.
- أي شيء داخل `${ }` يُنفَّذ كـ JavaScript وتُوضع نتيجته مكانه.

### 6.2 المفهوم الثاني: `.map()` — «حوّل كل عنصر»

`.map()` دالة على المصفوفات. تمرّ على كل عنصر وتُرجع **مصفوفة جديدة** ناتجة عن
تحويل كل عنصر. نستخدمها لتحويل كل كائن بيانات إلى قطعة HTML:

```js
DATA.nav.map((n) => `<li><a href="${n.href}">${n.label}</a></li>`)
```

- `(n) => ...` دالة تأخذ عنصرًا واحدًا (سمّيناه `n`) وترجع HTML الخاص به.
- لو كان `DATA.nav` فيه 7 روابط، تُرجع `.map` **مصفوفة من 7 نصوص HTML**:
  ```js
  ["<li><a href='#home'>الرئيسية</a></li>", "<li>...</li>", ... ]
  ```

### 6.3 المفهوم الثالث: `.join("")` — «الصق الكل في نصّ واحد»

`.map` تعطينا مصفوفة من النصوص، لكن `innerHTML` يحتاج **نصًّا واحدًا**. فنستخدم
`.join("")` لدمج عناصر المصفوفة في نصّ واحد بدون فواصل:

```js
["<li>أ</li>", "<li>ب</li>"].join("")
// النتيجة: "<li>أ</li><li>ب</li>"
```

> لو استخدمت `.join(",")` لظهرت فواصل بين العناصر. لذلك نستخدم `""` (نصّ فارغ).

### 6.4 المثال الكامل: بناء القائمة العلوية خطوة بخطوة

الكود الفعلي:

```js
setHTML("navList", DATA.nav.map((n) =>
  `<li><a href="${n.href}" class="${n.active ? "active" : ""}">${n.label}</a></li>`
).join(""));
```

لنفكّكه من الداخل للخارج:

1. **`DATA.nav`** = قائمة الروابط (7 كائنات).
2. **`.map((n) => ...)`** = لكل رابط `n`، أنشئ هذا الـ HTML:
   ```html
   <li><a href="#home" class="active">الرئيسية</a></li>
   ```
   - `${n.href}` ← يضع `#home`.
   - `${n.active ? "active" : ""}` ← لو `active=true` يضع كلمة `active`،
     وإلا يضع فراغًا (هذا يُسمّى **الشرط الثلاثي / ternary**، شرحه في القسم 6.5).
   - `${n.label}` ← يضع «الرئيسية».
3. **`.join("")`** = يلصق الـ 7 عناصر `<li>` في نصّ واحد طويل.
4. **`setHTML("navList", ...)`** = يضع هذا النصّ داخل `<ul id="navList">`.

**النتيجة النهائية في الصفحة:**

```html
<ul class="nav-list" id="navList">
  <li><a href="#home" class="active">الرئيسية</a></li>
  <li><a href="#collections" class="">المجموعات</a></li>
  <li><a href="#worlds" class="">العبايات</a></li>
  ... إلخ
</ul>
```

🎯 **هكذا تنتقل البيانات من JS إلى HTML:**
`DATA` (بيانات) → `.map` (تحويل لقوالب) → `.join` (دمج) → `setHTML/innerHTML`
(حقن) → تظهر في الصفحة → `CSS` ينسّقها.

### 6.5 الشرط الثلاثي (Ternary) `؟ :`

صيغة مختصرة لـ if/else داخل القالب:

```
الشرط ? القيمة_إذا_صحّ : القيمة_إذا_خطأ
```

أمثلة من الكود:
- `${n.active ? "active" : ""}` ← أضف صنف `active` للرابط النشط فقط.
- `${p.isNew ? '<span class="badge-new">NEW</span>' : ""}` ← أضف شارة «جديد»
  فقط للمنتج الذي عنده `isNew: true`.
- `${f.checked ? "checked" : ""}` ← فعّل صندوق الاختيار المحدّد مسبقًا.

### 6.6 حقن الأيقونات الثابتة (`data-icon`)

```js
document.querySelectorAll("[data-icon]").forEach((n) => {
  n.innerHTML = icon(n.dataset.icon);
});
```

- `querySelectorAll("[data-icon]")` ← يجلب **كل** عنصر فيه خاصّية `data-icon`.
- `.forEach((n) => ...)` ← يمرّ على كل عنصر `n`.
- `n.dataset.icon` ← يقرأ قيمة `data-icon` (مثل `"bag"`).
- `n.innerHTML = icon("bag")` ← يضع كود SVG داخل العنصر.

> **انتبه (نقطة مهمّة جدًّا):** هذا السطر يعمل **مرّة واحدة عند بدء التشغيل**،
> وقبل أن تُرسم القوائم المُولّدة بالـ JS. لذلك:
> - الأيقونات المكتوبة مباشرة في HTML بـ `data-icon` ← تُحقَن هنا تلقائيًّا. ✅
> - الأيقونات داخل قوالب الـ JS (البطاقات المُولّدة) ← تُكتب باستدعاء
>   `${icon("arrow")}` مباشرةً في القالب، لأنها تُبنى بعد تشغيل هذا السطر.

## 7. شرح كل دالة عرض (render) على حدة

كل الدوال تتبع نفس النمط: `setHTML("الحاوية", DATA.شيء.map(...).join(""))`.

### 7.1 الإحصائيات وشريط الثقة

```js
setHTML("heroStats", DATA.stats.map((s) =>
  `<div class="bar-stat"><span class="stat-num">${s.num}</span><span class="stat-label">${s.label}</span></div>`
).join(""));
```
لكل إحصائية: رقم (`s.num`) + وصف (`s.label`). بسيطة.

### 7.2 المجموعات (`collectionGrid`)

```js
setHTML("collectionGrid", DATA.collections.map((c, i) =>
  `<article class="col-card reveal" style="transition-delay:${i * 110}ms">
     <img src="${COVER}" alt="${c.title}" loading="lazy">
     <div class="col-body">
       <h3 class="col-title">${c.title}</h3>
       <p class="col-desc">${c.l1}<br>${c.l2}</p>
       <a href="#shop" class="col-cta"><span>تسوّقي الآن</span><span class="circle">${icon("arrow")}</span></a>
     </div>
   </article>`).join(""));
```

نقاط مهمّة هنا:
- **`(c, i)`**: لاحظ المعامل الثاني `i`. هذا **رقم ترتيب** العنصر (0، 1، 2، 3).
  نستخدمه في `transition-delay:${i * 110}ms` لجعل كل بطاقة تظهر بتأخير بسيط بعد
  التي قبلها (تأثير تتابعي جميل).
- **`src="${COVER}"`**: كل البطاقات تستخدم نفس صورة الغلاف `assets/cover.png`.
- **`${icon("arrow")}`**: تُحقَن أيقونة السهم مباشرة في القالب.
- **`loading="lazy"`**: تأجيل تحميل الصورة حتى تقترب من الشاشة (أسرع للموقع).

### 7.3 لماذا رزين (`whyGrid`)
نفس الفكرة: لكل عنصر أيقونة (`icon(w.ic)`) + عنوان (`w.t`) + وصف (`w.d`) + رقم
(`w.n`).

### 7.4 عوالم رزين (`worldsGrid`)
بطاقات على شكل أقواس. الصورة `${COVER}` تُقصّ بشكل القوس عبر CSS، وتحتها العنوان
والوصف وزر «استكشفي المجموعة».

### 7.5 المنتجات (`productRail`)

```js
${p.isNew ? '<span class="badge-new">NEW</span>' : ""}
...
<span class="product-dots">${p.dots.map((d) => `<span class="dot" style="background:${d}"></span>`).join("")}</span>
```
- شارة «NEW» تظهر بالشرط الثلاثي.
- **`.map` داخل `.map`** (متداخلة): لكل منتج، نمرّ على قائمة ألوانه `p.dots`
  ونبني نقطة لونية لكل لون. هذا يوضّح أن `.map().join("")` يمكن أن تتداخل.

### 7.6 الفلاتر

```js
setHTML("filterTypes", DATA.filterTypes.map((f) =>
  `<label class="filter-opt">
     <span>${f.label}</span>
     <input type="checkbox" ${f.checked ? "checked" : ""}>
     <span class="check-box">${icon("check")}</span>
   </label>`).join(""));
```
خيارات النوع (صناديق اختيار) وخيارات اللون (أزرار ملوّنة).

### 7.7 الفوتر — نمط مختلف (شرط على «النوع»)

الفوتر مميّز لأن أعمدته **ليست متطابقة**: عمود علامة تجارية، عمودان روابط، وعمود
نشرة بريدية. لذلك القالب يقرّر شكل العمود حسب `col.type`:

```js
setHTML("footerGrid", DATA.footer.map((col) => {
  if (col.type === "brand") {
    return `<div class="foot-col brand-col"> ... شعار وتواصل ... </div>`;
  }
  if (col.type === "news") {
    return `<div class="foot-col news"> ... نشرة بريدية ونموذج ... </div>`;
  }
  // الافتراضي = عمود روابط عادي:
  return `<div class="foot-col">
    <h4>${col.title}</h4>
    <div class="foot-links">${col.items.map((x) => `<a href="#">${x}</a>`).join("")}</div>
  </div>`;
}).join(""));
```

- هنا الدالة داخل `.map` **مطوّلة** (فيها `{ }` و `return`)، لأنها تحتوي منطق
  `if`. (الدوال القصيرة لا تحتاج `return`، لكن لو فتحت `{ }` فيجب أن تكتب
  `return` صراحةً.)
- `col.items.map(...)` ← تبني روابط العمود من قائمة `items`.

## 8. التفاعلات (الأزرار، القائمة، التمرير...)

بعد بناء الـ HTML، يأتي الجزء الثاني من `main.js`: تشغيل التفاعلات.

### 8.1 الهيدر اللاصق (يتغيّر عند التمرير)

```js
const header = el("siteHeader");
const onScroll = () => header.classList.toggle("scrolled", window.scrollY > 40);
onScroll();
window.addEventListener("scroll", onScroll, { passive: true });
```
- `window.scrollY` ← كم نزل المستخدم بالتمرير (بالبكسل).
- `classList.toggle("scrolled", شرط)` ← يضيف صنف `scrolled` لو الشرط صحيح
  (نزل أكثر من 40px)، ويزيله لو رجع للأعلى. الـ CSS يستخدم هذا الصنف لتغيير شكل
  الهيدر (خلفية/ظل).
- `addEventListener("scroll", ...)` ← «استمع لحدث التمرير ونفّذ الدالة».

### 8.2 قائمة الجوال (Hamburger)

```js
const setMenu = (open) => {
  nav.classList.toggle("open", open);
  overlay.classList.toggle("show", open);
  toggle.innerHTML = icon(open ? "close" : "menu");
  document.body.style.overflow = open ? "hidden" : "";
};
toggle.addEventListener("click", () => setMenu(!nav.classList.contains("open")));
overlay.addEventListener("click", () => setMenu(false));
nav.addEventListener("click", (e) => { if (e.target.closest("a")) setMenu(false); });
```
- `setMenu(true)` يفتح القائمة، `setMenu(false)` يغلقها.
- عند فتح القائمة: نغيّر أيقونة الزر إلى «إغلاق»، ونمنع تمرير الصفحة الخلفية
  (`overflow: hidden`).
- الضغط على زر القائمة **يبدّل** الحالة (`!...contains("open")` تعني «العكس»).
- الضغط على الخلفية المعتمة (overlay) أو على أي رابط يغلق القائمة.

### 8.3 اختيار اللون في الفلتر

```js
document.getElementById("filterColors").addEventListener("click", (e) => {
  const s = e.target.closest(".color-swatch");
  if (!s) return;
  document.querySelectorAll(".color-swatch").forEach((x) => x.classList.remove("active"));
  s.classList.add("active");
});
```
- **تفويض الحدث (Event Delegation):** بدل إضافة مستمع لكل زر لون، نضيف مستمعًا
  واحدًا على الحاوية. عند الضغط، `e.target` هو ما ضُغط عليه فعلًا، و
  `.closest(".color-swatch")` يصعد للأعلى ليجد زر اللون.
- `if (!s) return;` ← لو الضغط لم يكن على زر لون، توقّف.
- ننزع `active` عن كل الأزرار، ثم نضعه على المضغوط فقط (اختيار واحد).

### 8.4 سحب شريط المنتجات بالماوس (Drag-to-scroll)

```js
let down = false, startX = 0, startScroll = 0, moved = 0;
const start = (x) => { down = true; startX = x; startScroll = rail.scrollLeft; ... };
const move  = (x) => { if (!down) return; const d = x - startX; rail.scrollLeft = startScroll - d; };
const end   = () => { down = false; ... };
```
- عند الضغط (`mousedown`) نسجّل نقطة البداية.
- عند تحريك الماوس (`mousemove`) نحرّك الشريط بمقدار الإزاحة.
- عند الإفلات (`mouseup`) نتوقّف.
- `moved` يقيس مقدار السحب؛ لو سحبت كثيرًا، نلغي النقرة (`e.preventDefault()`)
  حتى لا يُفتح رابط بالخطأ أثناء السحب.

### 8.5 الظهور التدريجي عند التمرير (Scroll Reveal) ⭐

```js
const reveals = document.querySelectorAll(".reveal");
if ("IntersectionObserver" in window) {
  const io = new IntersectionObserver((entries) => {
    entries.forEach((en) => {
      if (en.isIntersecting) { en.target.classList.add("in"); io.unobserve(en.target); }
    });
  }, { threshold: 0.12, rootMargin: "0px 0px -8% 0px" });
  reveals.forEach((r) => io.observe(r));
} else {
  reveals.forEach((r) => r.classList.add("in"));
}
```
- أي عنصر صنفه `reveal` يبدأ مخفيًّا (شفّاف/منزاح، عبر CSS).
- **`IntersectionObserver`** أداة تراقب «هل دخل العنصر مجال الرؤية؟».
- عند دخول العنصر للشاشة (`isIntersecting`) نضيف صنف `in`، فيظهر بتأثير ناعم.
- `io.unobserve(...)` ← نتوقّف عن مراقبته بعد ظهوره (لا داعي لتكرار).
- `else` ← خطّة بديلة للمتصفّحات القديمة: أظهر كل شيء فورًا.

#### كيف يعمل `reveal` في الـ CSS؟ (الأساس المشترك)

كل عنصر `reveal` يبدأ مخفيًّا، ويظهر عند إضافة `in`:

```css
.reveal     { opacity: 0; transform: translateY(34px); transition: opacity .9s, transform .9s; }
.reveal.in  { opacity: 1; transform: none; }
```

أي: الجافاسكربت لا يحرّك شيئًا بنفسه، فقط **يضيف صنف `in`** في اللحظة المناسبة،
و**الـ CSS هو من ينفّذ الحركة** (الـ `transition`). هذا الفصل بين «المنطق» و«الشكل»
هو سرّ بساطة الكود.

### 8.6 حركة ظهور قسم «تسوّقي المجموعات» (انكشاف متتالٍ) ⭐

بطاقات المجموعات لها حركة ظهور **مخصّصة وأرقى** من الـ `reveal` العام: انكشاف
تدريجي من الأعلى (`clip-path`) + صعود + تكبير خفيف، وكل بطاقة تتأخّر قليلاً عن
سابقتها (تأثير متتالٍ / staggered).

**1) التأخير المتتالي يُحقن من الجافاسكربت** أثناء توليد البطاقات
([main.js](js/main.js))، عبر `transition-delay` يزيد مع رقم البطاقة `i`:

```js
DATA.collections.map((c, i) =>
  `<article class="col-card reveal" style="transition-delay:${i * 110}ms"> ... </article>`)
```

فالبطاقة الأولى تتحرّك فورًا (`0ms`)، الثانية بعد `110ms`، الثالثة `220ms`... وهكذا.

**2) الحركة نفسها معرّفة في الـ CSS** ([style.css](css/style.css)):

```css
/* الحالة المخفية: منزاحة لأسفل + مصغّرة + مقصوصة من الأعلى */
.col-card.reveal {
  opacity: 0;
  transform: translateY(54px) scale(0.97);
  clip-path: inset(16% 0 0 0 round var(--radius-md));   /* يُظهر 84% فقط، الباقي مقصوص */
  transition: opacity .9s, transform 1.05s, clip-path 1.05s;
}
/* الحالة النهائية بعد إضافة in: تستقر وتنكشف بالكامل */
.col-card.reveal.in {
  opacity: 1; transform: none;
  clip-path: inset(0 0 0 0 round var(--radius-md));
}
/* النص داخل البطاقة يصعد بالتوازي لإحساس بالعمق */
.col-card.reveal .col-body { opacity: 0; transform: translateY(24px); transition-delay: inherit; }
.col-card.reveal.in .col-body { opacity: 1; transform: none; }
```

- **`clip-path: inset(16% 0 0 0)`** ← يقصّ 16% من أعلى البطاقة، فتبدو وكأنها
  «تُكشف» تدريجيًّا حتى تكتمل عند `inset(0 0 0 0)`. كلمة `round var(--radius-md)`
  تحافظ على الزوايا المدوّرة أثناء القصّ.
- **`transition-delay: inherit`** على `.col-body` ← يجعل النص يرث نفس تأخير
  البطاقة، فيتزامن معها في التتالي.

**3) احترام تفضيل تقليل الحركة** — للمستخدمين الذين فعّلوا «تقليل الحركة» في
نظامهم، نُلغي الحركة ونُظهر البطاقات فورًا (مهم لإمكانية الوصول):

```css
@media (prefers-reduced-motion: reduce) {
  .col-card.reveal, .col-card.reveal .col-body { opacity: 1; transform: none; clip-path: none; transition: none; }
}
```

> **الخلاصة:** نفس آلية `reveal`/`in` العامّة (القسم 8.5) — الجافاسكربت يضيف `in`
> عند الظهور — لكن مع **CSS مخصّص** للبطاقات يعطيها حركة انكشاف فاخرة بدل التلاشي البسيط.

### 8.7 تتبّع القسم النشط في القائمة (Scroll-Spy)

```js
const spy = () => {
  const y = window.scrollY + window.innerHeight * 0.32;
  let cur = sections[0];
  for (const s of sections) { if (s.offsetTop <= y) cur = s; }
  navLinks.forEach((a) => a.classList.toggle("active", a.getAttribute("href") === "#" + cur.id));
};
```
- يحسب أي قسم يملأ الشاشة حاليًّا، ويضيف صنف `active` لرابط القائمة المقابل له،
  حتى يعرف المستخدم أين هو في الصفحة.

## 9. وصفات عملية: «كيف أعدّل...؟»

> القاعدة الذهبية: **99% من التعديلات تكون في كائن `DATA` داخل `js/main.js`،
> ولا تحتاج لمس `index.html` ولا `style.css`.**

### ✏️ تغيير اسم أو سعر منتج
في `js/main.js` → `DATA.products`، عدّل الكائن المطلوب:
```js
{ img: "prod-abaya-black", title: "عباية ليلك الحريرية", price: "450 ر.س", dots: ["#6b5640", "#181512"], isNew: true },
//                          ↑ غيّر العنوان            ↑ غيّر السعر
```

### ➕ إضافة منتج جديد
أضف كائنًا جديدًا في نهاية مصفوفة `DATA.products` (لا تنسَ الفاصلة `,`):
```js
{ img: "prod-new", title: "عباية جديدة", price: "500 ر.س", dots: ["#000000"], isNew: true },
```
ثم ضع صورة باسم `prod-new.jpg` داخل `assets/images/`.

### ➕ إضافة/تعديل رابط في القائمة العلوية
في `DATA.nav`:
```js
{ label: "تواصلي معنا", href: "#visit" },
```
- `label` = النصّ الظاهر، `href` = القسم الذي ينتقل إليه (يطابق `id` القسم).

### ✏️ تعديل نصوص الفوتر
في `DATA.footer`، عدّل `title` أو عناصر `items`:
```js
{ type: "links", title: "خدمة العملاء", items: ["الشحن والتوصيل", "الإرجاع والاستبدال", ...] },
```

### 🖼️ تغيير صورة بطاقات المجموعات/العوالم
كلها تستخدم الثابت `COVER`. غيّر سطرًا واحدًا في `js/main.js`:
```js
const COVER = "assets/cover.png";  // ← ضع اسم صورة أخرى
```
ولو أردت صورة مختلفة لكل بطاقة، استبدل في القالب `${COVER}` بـ `${img(c.img)}`
واضبط حقل `img` في البيانات.

### 📊 تغيير أرقام الإحصائيات
في `DATA.stats`:
```js
{ num: "4.9", label: "تقييم المتجر" },  // ← غيّر num و label
```

### 🗺️ تغيير موقع الخريطة (في قسم «موقعنا»)
الخريطة مكتوبة مباشرة في `index.html` (قسم VISIT US) كـ `<iframe>`. غيّر
الإحداثيات في الرابط:
```
https://maps.google.com/maps?q=35.9306,36.6339(...)&z=16&hl=ar&output=embed
                              ↑ خط العرض ↑ خط الطول        ↑ التكبير
```
وللحصول على إحداثيات: افتح Google Maps، اضغط يمين على المكان، وانسخ الرقمين.

### 🎨 إضافة أيقونة جديدة
في `js/main.js` → كائن `I`، أضف مفتاحًا جديدًا بكود SVG:
```js
phone: '<svg viewBox="0 0 24 24" ...>...</svg>',
```
ثم استدعها في أي بيانات بـ `ic: "phone"` أو في HTML بـ `data-icon="phone"`.

## 10. مسرد المصطلحات البرمجية

| المصطلح | المعنى المبسّط |
|---|---|
| **DOM** | شجرة عناصر الصفحة كما يفهمها المتصفّح |
| **`const`** | متغيّر لا تتغيّر قيمته بعد تعريفه |
| **`let`** | متغيّر يمكن تغيير قيمته (مثل `down`, `startX`) |
| **Object `{}`** | كائن: مجموعة خصائص (مفتاح: قيمة) |
| **Array `[]`** | مصفوفة: قائمة مرتّبة من العناصر |
| **Arrow function `() => {}`** | دالة مختصرة |
| **Template literal `` ` ` ``** | نصّ يقبل متغيّرات بـ `${ }` |
| **`.map()`** | تحوّل كل عنصر في مصفوفة وتُرجع مصفوفة جديدة |
| **`.join("")`** | تدمج مصفوفة نصوص في نصّ واحد |
| **`.forEach()`** | تمرّ على كل عنصر دون إرجاع شيء |
| **Ternary `? :`** | if/else مختصر داخل تعبير |
| **`innerHTML`** | محتوى HTML داخل عنصر (قراءة/كتابة) |
| **`getElementById`** | يجلب عنصرًا بالـ id |
| **`querySelectorAll`** | يجلب كل العناصر المطابقة لمُحدِّد CSS |
| **`addEventListener`** | يربط حدثًا (نقر/تمرير) بدالة |
| **`classList.toggle/add/remove`** | يتحكّم بأصناف CSS على عنصر |
| **`dataset`** | يقرأ خصائص `data-*` من عنصر |
| **IIFE** | دالة تُنفَّذ فور تعريفها وتعزل الكود |
| **IntersectionObserver** | يراقب دخول العناصر لمجال الرؤية |
| **lazy loading** | تأجيل تحميل الصور حتى الحاجة |



### خلاصة في ٣ أسطر

- **`index.html`** = هيكل ثابت + «علب فارغة» لها `id`.
- **`DATA` في `main.js`** = كل المعلومات القابلة للتغيير (هنا تعدّل غالبًا).
- **دوال العرض** تأخذ `DATA` → `.map()` تبني قوالب → `.join("")` تدمجها →
  `setHTML()` تحقنها في العلب الفارغة → **يظهر الموقع**.
